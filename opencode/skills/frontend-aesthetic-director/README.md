# Frontend Aesthetic Director

Repo-managed implementation-facing skill for frontend UI polish, existing-surface cleanup, visual direction, responsive behavior, accessibility states, and rendered QA.

Canonical skill name: `frontend-aesthetic-director`

## Summary

Use this skill when visible frontend code is being implemented or reviewed:
- landing pages and marketing sections
- dashboard and admin polish
- components, forms, data tables, and settings screens
- existing UI polish, visual cleanup, and anti-slop passes
- screenshot, Figma, or `/uiux` handoff implementation
- responsive and accessibility state cleanup
- implementation of approved redesigns that should preserve the current flow

## Recommended Pairing

- Upstream concept: `/uiux` for wireframes, flow, communication, and conceptual handoff
- Implementation: `frontend-aesthetic-director` for tokens, hierarchy, components, and responsive UI code
- Evidence: browser tooling, Playwright, screenshots, or `devtools-ux-audit` when rendered validation is needed
- Not a fit: backend-only changes, conceptual-only UI output, screenshot/wireframe critique before implementation, or formal UX scorecard generation

## Files

- `SKILL.md`: main skill definition
- `references/layout-style-playbook.md`: layout archetypes, visual style profiles, and polish-first correction strategies
- `references/polish-checklist.md`: anti-slop guardrails, state audit, responsive audit, and pre-flight checks
- `references/ui-quality-rubric.md`: visual and usability review rubric
- `assets/design-brief-template.md`: compact polish brief template with preserve-vs-modernize and risk fields

## One-Line Positioning

Preserve the approved flow, then make the implemented UI intentional, responsive, accessible, and visibly less generic.
