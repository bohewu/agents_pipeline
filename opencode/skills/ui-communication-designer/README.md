# UI Communication Designer

This is a docs-only skill package derived from the core ideas in Everett N. McKay's *UI is Communication*.

Canonical skill name: `ui-communication-designer`

## Summary
This skill treats UI as task-oriented communication rather than simple component arrangement.  
It is especially useful for:
- designing task flows from requirements
- reviewing existing screenshots or wireframes
- rewriting UI microcopy
- checking forms, settings pages, checkout, onboarding, and error flows
- diagnosing UI problems in concrete, non-aesthetic terms

## Recommended Pairing
- Best fit: `/uiux` for conceptual design, flow reframing, and screen or copy rewrite work
- Secondary fit: run `/run-ux` first for audit findings, then feed those findings into this skill for a communication-first redesign
- Not a fit: browser-backed audits, viewport scoring, or implementation-ready specs

## Files
- `SKILL.md`: main skill definition
- `RUBRIC.md`: scoring and review rubric
- `OUTPUT_TEMPLATE.md`: standard output format

## One-Line Positioning
Design the conversation before designing the screen.
