# UI Communication Review Rubric

Each dimension is scored from 0-2.  
0 = clearly problematic  
1 = usable but unstable  
2 = clear, stable, and predictable

This rubric is aligned with `SKILL.md` and `OUTPUT_TEMPLATE.md`. Total score: 24.

## 1. Value Clarity
- Can the user quickly understand what value this screen or flow provides?
- Does it show outcomes and benefits before system mechanics?

## 2. Task Clarity
- Does the user know what to do right now?
- Does the primary instruction sound like something a person would naturally say?

## 3. Discoverability
- Is the starting point easy to find?
- Is the main command visible and placed where users expect it?

## 4. Understandability
- Can the user understand it quickly on first view?
- Does it avoid internal terms and unnecessary abbreviations?

## 5. Affordance
- Do components clearly communicate how they can be used?
- Is important text placed on labels instead of buried in instructions when possible?

## 6. Predictability
- Can users predict the result before acting?
- Is the commit point obvious?

## 7. Efficiency
- Does it avoid unnecessary login, setup, repeated input, confirmation, and extra steps?
- Does it prefer not asking, auto-inference, sensible defaults, and deferring questions when possible?

## 8. Feedback
- Is feedback timely, specific, and low-disruption?
- Do errors identify the object, the problem, and the next step?

## 9. Trust
- Is the interface specific, transparent, and verifiable?
- Does it avoid hidden cost, vague risk, and late disclosure of important information?

## 10. Forgiveness
- Does it prevent common mistakes?
- When mistakes happen, is recovery easy and is prior input preserved when possible?

## 11. Non-Annoyance
- Does it avoid unnecessary interruption, repetition, noise, and distraction?
- Does it interrupt only when necessary?

## 12. Scanability
- Is the focal point clear?
- Is the eye path natural?
- Is important content on the scan path instead of buried in long text?

## Quick Interpretation

- 0-11: the flow or information architecture likely needs redesign
- 12-17: there are major structural issues; copy edits alone are not enough
- 18-21: the structure is usable but still needs several fixes
- 22-24: the experience is broadly clear and stable; focus on refinement

If the total score is below 18/24, propose structural changes rather than copy-only edits.
